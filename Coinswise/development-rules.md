# CoinsWise Development Rules

These rules apply to every file in the project.

## HTML

- Semantic HTML5 only.
- One section per logical component.
- Accessible labels where applicable.
- No inline styles.
- No inline JavaScript.

## CSS

- Use CSS variables.
- Mobile-first responsive design.
- BEM-inspired class names where appropriate.
- One responsibility per stylesheet.

## JavaScript

- Modular.
- No global variables unless necessary.
- Descriptive function names.
- Event delegation where appropriate.

## Git

Commit after every completed feature.

Commit format:

feat:
fix:
docs:
style:
refactor:
perf:
